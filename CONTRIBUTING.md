## Quick Start for Contributors

### 1. Clone the Repository
```bash
git clone https://github.com/RecurPixel/RecurPixel.EasyMessages.git
cd RecurPixel.EasyMessages
```

### 2. Build the Project
```bash
dotnet build
```

### 3. Run Tests
```bash
dotnet test
```

### 4. Try the Samples
```bash
# Console sample
cd samples/ConsoleSample
dotnet run

# Web API sample
cd samples/WebApiSample
dotnet run
```

## Development Guidelines

### Code Style
- Follow existing code conventions
- Use meaningful variable and method names
- Add XML documentation comments for public APIs
- Keep methods focused and concise

### Testing
- Write unit tests for new features
- Ensure all existing tests pass
- Aim for high code coverage
- Test edge cases and error conditions

### Pull Requests
- Create a feature branch from `main`
- Write clear, descriptive commit messages
- Reference related issues in your PR description
- Ensure all tests pass before submitting
- Keep PRs focused on a single feature/fix

### Documentation
- Update relevant wiki pages for new features
- Add code examples where applicable
- Update CHANGELOG.md for your changes
- Ensure all code examples compile
